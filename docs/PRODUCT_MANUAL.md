# TsekTxt Product Manual

**Version:** 1.1
**Project:** TsekTxt — An Explainable Web-Based Taglish Credibility Checker
**Institution:** Adamson University, BS Data Science Capstone

---

## 1. Introduction

TsekTxt is a web application that helps Filipino users assess the credibility of Taglish (Tagalog-English) social media content. Users can submit text or upload screenshots, and the system returns a real-time credibility verdict powered by fine-tuned transformer models. The tool is designed as a credibility aid — it supports critical thinking before believing or sharing content, but does not replace professional fact-checking.

## 3. Getting Started

### 3.1 Creating an Account

1. Open the app at `http://localhost:3000` (or the deployed URL)
2. Click **Sign in** in the navigation bar
3. Choose one of two options:
   - Click **Continue with Google** to sign in using your Google account
   - Enter your email and password, then click **Sign in**
4. If you don't have an account, click **Create an account** and fill in your name, email, and password

### 3.2 Signing Out

- From the dashboard, click **Sign out** in the left sidebar at the bottom
- Or go to **Settings** and click **Sign out**

## 4. Using the Checker

### 4.1 Text Analysis

1. Navigate to the **Checker** page from the dashboard sidebar
2. Make sure the **Text** tab is selected
3. Paste or type a Taglish social media post, caption, or forwarded message into the text area
4. The text must be at least 50 characters
5. Click **Run suspiciousness check**
6. Wait for the analysis to complete (10-15 seconds on local CPU)
7. Review the results in the panel on the right

### 4.2 Image / Screenshot Analysis

1. On the Checker page, click the **Image** tab
2. Either drag and drop a screenshot into the upload area, or click **Choose image**
3. Supported formats: PNG, JPG, JPEG
4. The system will extract text from the image using Groq Vision (OCR)
5. The extracted text is then analyzed and shown in the editor
6. Review the results in the results panel

### 4.3 Loading a Sample

- Click **Load sample** to fill the text area with a pre-labeled example
- This is useful for testing the system without having your own content ready

### 4.4 Clearing Results

- Click the **Clear** button (trash icon) to reset the text area and results panel
- This prepares the checker for a new analysis

## 5. Understanding the Results

### 5.1 Verdict

The verdict is the primary output of the XLM-RoBERTa model:

| Verdict | Meaning |
|---|---|
| Not Suspicious | The text shows credibility signals — factual language, official references, no urgency markers |
| Suspicious | The text contains misinformation signals — hearsay, urgency, unverified claims |

### 5.2 Confidence Score

The confidence percentage shows how strongly the model believes in its verdict. A score of 95% means the model is highly confident. Lower scores suggest mixed signals in the text.

### 5.3 Highlighted Tokens

The results panel shows specific words and phrases highlighted in the text. These are the tokens that most influenced the model's decision, identified using Integrated Gradients attribution from the Captum library.

- Red highlights indicate tokens that raised suspicion
- Green highlights indicate tokens that reduced suspicion
- Click any highlighted phrase to see the explanation

### 5.4 Plain-Language Summary

Below the verdict, a short 1-2 sentence explanation (generated by Groq LLaMA 3.3) explains why the model classified the text this way in plain English. This does not change the verdict — it only explains it.

### 5.5 Model Comparison

The **Advanced model comparison** section shows the classification scores from all three models:
- XLM-RoBERTa (TsekTxt) — primary classifier
- RoBERTa-Tagalog — Filipino-specific model
- mBERT — multilingual BERT model

This lets you compare how each model scored the same text.

### 5.6 Responsible Use Notice

Every result includes a disclaimer reminding users that TsekTxt is a credibility aid, not a definitive fact-checker. Always verify suspicious content with official sources like COMELEC, Rappler Fact Check, or Vera Files before sharing.

## 9. System Architecture (For Developers)

### Components

| Component | Technology | Purpose |
|---|---|---|
| Frontend | Next.js 16, TypeScript, Tailwind CSS | User interface |
| Authentication | Supabase Auth | Email/password and Google OAuth |
| Text/Image API | Next.js API Routes | Orchestrates model + Groq calls |
| ML Backend | FastAPI (Python, port 8000) | Serves the 3 transformer models |
| Primary Model | chimsio/tsektxt-xlmr | XLM-RoBERTa fine-tuned on Filipino misinformation |
| Comparison Models | chimsio/tsektxt-roberta-tagalog, chimsio/tsektxt-mbert | Additional model scores |
| Explainability | Captum (Integrated Gradients) | Token attribution |
| Language Interpretation | Groq LLaMA 3.3 70B | Plain-language summary |
| OCR | Groq Vision (llama-4-scout) | Text extraction from images |

### Starting the System

Terminal 1 — FastAPI backend:
```zsh
cd fastapi-backend
uvicorn main:app --port 8000
```

Terminal 2 — Next.js frontend:
```zsh
corepack pnpm dev
```

### Environment Variables

Located at `.env.local` in the project root:

| Variable | Purpose |
|---|---|
| `NEXT_PUBLIC_USE_MOCK_API` | Set to `false` for real analysis |
| `GROQ_API_KEY` | Groq API key for LLaMA and Vision |
| `FASTAPI_URL` | FastAPI backend URL (default: `http://127.0.0.1:8000`) |
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase anonymous key |

## 10. Limitations

- The model was trained primarily on Philippine election misinformation data. Accuracy may vary for other domains.
- Image OCR works best on clear, readable screenshots. Stylized or low-resolution images may extract text inaccurately.
- Analysis takes 10-15 seconds on local CPU. Cloud deployment significantly reduces this.
- The tool is a research prototype and should not be used as the sole basis for fact-checking decisions.
- Google sign-in redirect requires the callback URL `http://localhost:3000/auth/callback` to be registered in both Supabase and Google OAuth settings.

---

## 11. Troubleshooting

| Problem | Solution |
|---|---|
| "Mock mode active" warning | Set `NEXT_PUBLIC_USE_MOCK_API=false` in `.env.local` |
| "Could not reach model server" | Start FastAPI: `uvicorn main:app --port 8000` in `fastapi-backend/` |
| Google sign-in loops back to sign-in page | Add `http://localhost:3000/auth/callback` to Supabase Redirect URLs |
| Port 3000 already in use | Run `kill $(lsof -ti:3000)` then restart |
| Stale build errors | Run `rm -rf .next` then `corepack pnpm dev` |
