## Title
TsekTxt: An Explainable Web-Based Taglish Credibility Checker for Filipino Social Media Content (Text and Image Classification)

---

## Introduction

TsekTxt is a web-based Taglish credibility checker developed as a capstone and thesis project for the BS Data Science program at Adamson University. The system lets Filipino users submit Taglish (Tagalog-English code-switched) social media text — typed directly or extracted from an uploaded screenshot via Groq multimodal vision (OCR) — and receive a real-time two-class credibility classification (Not Suspicious or Suspicious) with a confidence score.

The verdict and confidence are produced by the fine-tuned XLM-RoBERTa model (Hugging Face: chimsio/tsektxt-xlmr), a fine-tuned version of xlm-roberta-base trained on a combined Filipino misinformation corpus (Fake News Filipino Dataset and Philippine Fake News Corpus), served through a FastAPI backend inference endpoint on port 8000. Each result includes token-level Integrated Gradients attribution highlighting the words that most influenced the decision, and a plain-language interpretation generated by Groq LLaMA 3.3 (which explains the model's output only and does not decide the verdict).

This plan validates the system's features, usability, reliability, performance, and security against ISO 25010 using the FURPS model before final submission.

## Approach

Black Box Testing is the primary method used to validate TsekTxt. Testers interact with the system exclusively through its user interface, focusing on inputs and expected outputs without knowledge of the internal model architecture. Each feature is tested by submitting defined inputs and verifying the system returns the correct responses specified in the test cases.

For classification, sample Taglish content from the held-out labeled test data is used to verify the expected verdicts produced by the XLM-RoBERTa model. Image analysis is tested by uploading screenshots and confirming the OCR-extracted text is classified. For explainability, testers confirm that the Integrated Gradients attribution highlights the most influential tokens.

A grey-box cross-check is performed by calling the FastAPI inference endpoint (port 8000) directly to validate raw model output against the displayed result. The classifier is validated against an accuracy target of at least 90% (>= 0.90 macro-F1) on a labeled benchmark set; the fine-tuned chimsio/tsektxt-xlmr model reports 0.97 accuracy on its held-out test split. Accessibility is assessed against WCAG 2.1 AA. Performance testing measures API response time against the defined benchmark.

## Item Pass/Fail Criteria

| Component | Pass | Fail |
|---|---|---|
| UI/UX Design | All pages render correctly. Navigation links route to the correct pages. Layout is responsive on desktop and mobile viewports. | Pages fail to render or show broken layouts. Navigation links lead to wrong pages or 404 errors. |
| Text Analysis | Submitted Taglish text returns a verdict (Not Suspicious / Suspicious) derived from the XLM-RoBERTa model with a confidence score. | System returns no verdict, an error, or fails to respond. |
| Image Analysis (OCR) | An uploaded screenshot is processed by Groq vision, the extracted text is passed to the XLM-RoBERTa classifier, and a verdict with confidence is returned. | Image upload fails, no text is extracted, or the extracted text is not classified. |
| Model Accuracy | The classifier achieves at least 90% accuracy and >= 0.90 macro-F1 on the labeled benchmark set. | Accuracy or macro-F1 falls below the 90% / 0.90 target. |
| Results Display | Results panel shows the correct verdict label and a confidence percentage between 0% and 100%. | Results panel is blank, shows incorrect labels, or omits the confidence score. |
| Explainability (Integrated Gradients) | At least 3 tokens are highlighted, corresponding to the highest Integrated Gradients attribution weights from the XLM-RoBERTa model. | Token highlights are absent, random, or not derived from the model's Integrated Gradients attribution. |
| Authentication | Users can register, log in, and log out. Invalid credentials show an error. Expired sessions force re-authentication and concurrent logins behave per the session policy. | Valid credentials are rejected, invalid credentials are accepted, sign-out does not clear the session, or expired sessions still access protected pages. |
| Accessibility | Key pages meet WCAG 2.1 AA: full keyboard navigation with visible focus, labeled controls / ARIA names, and text contrast ratio >= 4.5:1. | Pages are not keyboard-navigable, controls lack accessible names, or text contrast falls below 4.5:1. |
| History | Each completed analysis is logged with the correct text, verdict, confidence, and timestamp. | Analyses are not logged, entries are missing fields, or history is inaccessible. |
| Error Handling | Empty submissions display a validation message. The system does not crash on invalid inputs. | Empty submissions are processed without error, or the system crashes / returns unhandled exceptions. |
| Performance & Reliability | FastAPI + XLM-RoBERTa inference responds within 15 seconds (local CPU). Identical inputs yield consistent verdicts (deterministic inference). | Text analysis exceeds 15 seconds or times out, or identical inputs yield inconsistent verdicts. |

## Suspension Criteria and Resumption Requirements

Testing is suspended when a critical defect blocks the core classification pipeline — such as the FastAPI backend being unreachable, the fine-tuned model checkpoint (chimsio/tsektxt-xlmr) failing to load or serve predictions, the Groq vision OCR failing to extract text, the Integrated Gradients module failing, or the results panel failing to render verdicts. All test cases dependent on the affected component are suspended. Testing resumes once the defect has been diagnosed, fixed, and the affected component retested in isolation and confirmed working.
