# UI Research and Inspiration

## Research goal

Use familiar writing-assistant patterns to reduce learning time, but do not clone any product’s visual identity, assets, wording, or exact layout.

## 1. Grammarly

Sources:

- https://www.grammarly.com/
- https://www.grammarly.com/ai-detector

Useful patterns:

- One dominant input action with a clear result beside it.
- The user sees a percentage and explanatory copy, not only a label.
- Product limitations are stated plainly.
- The editor remains the main workspace rather than becoming a dense dashboard.
- Result details can be progressively disclosed.

Adaptation for this project:

- Replace writing correction with credibility analysis.
- Show a confidence estimate and model agreement.
- Keep the warning that automated analysis is not conclusive.
- Highlight influential phrases directly inside the submitted text.

Do not copy:

- Grammarly green, logo, typography treatment, wording, cards, or illustrations.

## 2. QuillBot

Sources:

- https://quillbot.com/
- https://quillbot.com/ai-content-detector

Useful patterns:

- A suite-like application shell with quick access to tools and projects.
- Paste/type/upload as immediate entry points.
- Clear 0–100 scoring and section-level feedback.
- Explanations attached to flagged passages.
- History, reports, and adjacent tools are secondary to the editor.

Adaptation for this project:

- Use one focused checker rather than a large suite of unrelated tools.
- Keep history in a lightweight sidebar or dashboard.
- Use line- or phrase-level explanations.
- Put technical model details behind an “Advanced analysis” disclosure.

Do not copy:

- QuillBot green, feather mark, toolbar structure, exact navigation, or content hierarchy.

## 4. Behance exploration terms

Behance projects vary and are often presented as long case studies. Search these exact terms when doing a second visual pass:

- `AI writing assistant dashboard UI UX`
- `fact checking app UI UX`
- `text analysis SaaS dashboard`
- `misinformation detection app design`
- `media literacy mobile app UI`

Extract only high-level patterns: onboarding, trust signals, results visualization, editor layout, and responsive behavior. Do not copy complete screens.

## 5. Recommended synthesis

The frontend should combine:

- Grammarly’s focused editor and straightforward result summary;
- QuillBot’s section-level explanations and history-oriented product shell;
- Dribbble’s modern card spacing and desktop three-zone layout;
- an original, politically neutral Filipino brand system.

## 6. Anti-patterns

- Generic purple gradients on every surface.
- Excessive glassmorphism.
- Fake metrics, logos, testimonials, or accuracy claims.
- Red/green-only status communication.
- A result that appears legally or scientifically conclusive.
- Dense charts before the user sees the plain-language answer.
- Decorative animation that makes the checker feel less serious.
- Copying Grammarly or QuillBot closely enough to look derivative.
